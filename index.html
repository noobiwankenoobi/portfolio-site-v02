pathfinder algorithm visualizer

project description

this is the best coding i have done to date

this project was created with vanilla javascript with a little jquery,
and vanilla css animations

i finished it over the course of three days

i wanted to learn algorithms again because i had learned search algorithms in my discrete math class when i was 18 and had forgotten a lot of what i had learned

i also wanted to pass coding interviews

i thought that making a project that helped me visualize what was happening in different pathfinder algorithms would be the best way to learn them, as it would force me to code them properly and test my css skills

version 01 of the project visualizes dijkstra's search algorithm by highlighting visited nodes in blue on an interval and drawing the shortest path between the starting and finishing node

it allows users to click on the grid and make certain nodes a "wall" that the algorithm has to find a way around

version 02 of this project will add two or three more types of pathfinder algorithms, and possibly allow the user to add "weighted" nodes in addition to wall nodes

woofer - full stack twitter clone

project description

This is a full stack twitter clone project called "woofer

it's twitter for dogs

you type in your dog's name and what they would say, and it makes a post and displays it on the screen

i made two separate back ends for this project

one with express, node, monk, and mongo, and one with rails and postgresql

because i wanted practice with both

in the next version, i will add auth so that users can log in and toggle between seeing all woofs or only their own woofs

then they also won't have to type in a name

the version after that will allow users to friend other users and only see woofs from their friends if they want

face detection - real-time in browser

project description

i implemented an existing wrapper for tensor flow called face api js

by leveraging this api i was able to get real-time face detection in the browser that "reads" expressions and displays them

i utilized async functions, intervals, and had to read api documentation

i also figured out how to implement user media (the webcam) and the video streaming object

in the next version, i add more detections other than expressions, like gender, etc

i would like to change the display based on what user emotion is being detected
